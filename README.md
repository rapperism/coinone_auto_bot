# 코인원 자동매매 봇

## 📦 설치

```bash
pip install -r requirements.txt
```

## 🔑 API 키 설정

1. 코인원 웹사이트 로그인
2. 하단 footer > Open API > 통합 API 관리 > 개인용 API
3. [새로운 키 발급] 클릭
4. 권한 선택: **잔고 조회 + 주문 권한** 체크
5. `.env.example` → `.env` 로 복사 후 키 입력

```bash
cp .env.example .env
# .env 파일 열어서 토큰/시크릿 입력
```

## ⚙️ 설정 변경 (bot.py 상단 CONFIG 딕셔너리)

| 항목 | 기본값 | 설명 |
|------|--------|------|
| SYMBOL | btc | 거래 코인 (btc/eth/xrp 등) |
| ORDER_RATIO | 0.3 | 원화 잔고의 30%씩 매수 |
| STOP_LOSS_PCT | 0.03 | 3% 손절 |
| TAKE_PROFIT_PCT | 0.05 | 5% 익절 |
| TAKER_FEE_PCT | 0.02 | 체결 수수료 0.02%. 손익분기·수수료 반영 손익에 사용 |
| LOOP_INTERVAL | 60 | 60초마다 신호 체크 |
| CANDLE_INTERVAL | 1m | 1분봉 기준 |
| VOL_K | 0.5 | 변동성 돌파 K값 |
| VOL_BREAKOUT_USE_DAILY | false | true면 변동성 돌파 목표가를 일봉(전일 고·저+당일 시가)으로 계산 |
| RELAX_CROSS_SIGNALS | false | true면 골든/데드·MACD를 "순간"이 아닌 "상태 유지"로 인정 (BUY 빈도 증가) |

## 📊 전략 설명

### 복합 신호 방식 (5개 지표 조합)

**기본:** 캔들 `CANDLE_INTERVAL`(기본 1m) 기준으로 직전 봉/현재 봉을 사용합니다.

**매수 신호** — 아래 중 3개 이상 충족 시 매수
1. 이동평균 골든크로스 (단기 EMA5 > 장기 EMA20, **교차 발생한 그 봉**에서만 인정)
2. RSI 과매도 (RSI < 30)
3. 변동성 돌파 (현재가 > 목표가). 목표가는 `VOL_BREAKOUT_USE_DAILY`가 False면 **직전 봉** 시가+범위×K, True면 **일봉** 전일 고·저+당일 시가×K (클래식)
4. MACD 히스토그램 양전환 (음→양, **전환된 그 봉**에서만 인정)
5. 볼린저밴드 하단 반등 (직전 봉 종가 ≤ 하단밴드, 현재가 > 하단밴드)

**매도 신호** — 아래 중 2개 이상 충족 시 매도
1. 이동평균 데드크로스 (단기 < 장기, 교차한 그 봉만)
2. RSI 과매수 (RSI > 70)
3. MACD 히스토그램 음전환 (양→음, 전환된 그 봉만)
4. 볼린저밴드 상단 되돌림

**신호 빈도가 너무 낮을 때**  
골든/데드크로스·MACD 전환이 “그 순간”에만 인정되어 BUY가 며칠씩 안 나올 수 있습니다. `CONFIG`에서 `RELAX_CROSS_SIGNALS: True`로 두면, **단기>장기 유지**·**MACD 히스토그램 양수 유지**도 1점으로 인정해 매수 신호가 더 자주 나옵니다. (자세한 검증은 `STRATEGY_REVIEW.md` 참고)

**리스크 관리 (전략 신호보다 우선)**
- 손절: 진입가 대비 -3% 시 즉시 매도
- 익절: 진입가 대비 +5% 시 즉시 매도
- **수수료 반영**: 코인원 API 수수료(기본 0.02%)를 감안한 손익분기가 계산됩니다. 현재가가 손익분기(매수금+수수료 회수 가능 가격) 미만이면 **매도하지 않고 보류**해, 통장 기준 손실 매도를 막습니다. 손익 로그도 수수료 반영 금액/수익률로 표시됩니다.
